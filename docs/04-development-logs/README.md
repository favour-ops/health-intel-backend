## Getting Started with Logs

When you complete work:

1. Create a new file: `[date]-[short-title].md`
   - Example: `2025-01-23-hospital-crud-complete.md`

2. Fill in the template above

3. Include actual numbers and concrete details

4. Link to PRs, issues, docs

5. Be honest about challenges

## Why This Matters

These logs become:
- **Blog posts** (polished versions)
- **Onboarding docs** (for new team members)
- **Portfolio pieces** (showing your process)
- **Learning resources** (for students)

Don't think "internal only." Think "publishable."
